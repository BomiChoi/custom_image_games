# 커스텀 이미지 게임 — 프로젝트 개요

## 프로젝트 소개

사용자가 직접 업로드한 이미지로 게임을 즐기고, 설정한 테마를 다른 사람과 공유할 수 있는 웹 서비스입니다.

---

## 주요 기능

### 게임

| 게임 | 이미지 설정 | 설명 |
|---|---|---|
| **2048** | 최대 16개 | 타일 단계(2, 4, 8 … 65536)마다 개별 이미지 지정 |
| **수박게임** | 최대 16개 | 병합 레벨(체리~수박 11단계)마다 개별 이미지 지정 |
| **슬라이드 퍼즐** | 1개 | 단일 이미지를 3×3 또는 4×4 격자로 분할하여 퍼즐 구성 |

- 이미지를 설정하지 않은 슬롯은 기본 색상 스타일로 표시되므로, 일부 슬롯만 설정해도 플레이 가능

### 테마 저장 및 공유

- 이미지 설정을 **테마**로 저장하고 고유 링크(`/share/{8자리코드}`) 생성
- **공개**: 사이트 탐색 페이지에 노출, 누구나 찾아볼 수 있음
- **링크 공유**: URL을 아는 사람만 접근 가능, 탐색 페이지에 미노출

### 리더보드

- 게임 종료 후 닉네임 입력 → 점수 등록
- **이 테마 순위**: 같은 테마(이미지 설정) 내에서의 랭킹
- **전체 순위**: 해당 게임 타입 전체 랭킹
- 기간 필터: 전체 / 이번 주 / 이번 달

## 설계 결정

### 비회원 구조
별도 회원가입 없이 닉네임만으로 점수를 등록합니다. 간편하게 참여할 수 있고, 테마 제작 시 닉네임을 입력하면 제작자로 표시됩니다.

### 점수 위조 방지
클라이언트가 점수를 직접 DB에 쓸 수 없도록 서버 API에서만 점수를 저장합니다. 게임 시작 시 서버가 세션 토큰을 발급하고, 종료 시 HMAC 서명 검증 및 최소 플레이 시간 체크를 거칩니다.

### 이미지 처리
업로드한 이미지는 클라이언트에서 먼저 1024×1024로 리사이즈·WebP 변환 후 서버로 전송합니다. 슬라이드 퍼즐은 이미지를 분할하지 않고 CSS `background-position` 트릭으로 각 조각을 표현하여 네트워크 요청을 1회로 줄입니다.

### 임시 업로드 경로
이미지 업로드와 테마 저장이 별개 단계로 분리됩니다. 업로드 시 `tmp_{id}` 임시 경로에 저장 후, 테마 생성 시 확정 경로로 이동합니다. 이를 통해 미완성 업로드가 DB에 남지 않습니다.

### 테마 만료
테마는 생성 후 30일 뒤 만료(`expires_at`)됩니다. 만료된 테마는 조회되지 않으며, 주기적인 정리 함수(`cleanup_expired_records`)를 통해 삭제됩니다.

## 화면 구성

```
/                        홈 — 게임 선택 + 공개 테마 탐색
/games/2048              2048 게임 (기본 스타일)
/games/2048/{shortId}    2048 게임 (커스텀 테마 적용)
/games/suika             수박게임 (기본 스타일)
/games/suika/{shortId}   수박게임 (커스텀 테마 적용)
/games/slide-puzzle      슬라이드 퍼즐 (기본 스타일)
/games/slide-puzzle/{shortId}  슬라이드 퍼즐 (커스텀 테마 적용)
/share/{shortId}         테마 공유 랜딩 — 미리보기 + 게임 시작 버튼
/leaderboard             전체 리더보드 (게임별 탭)
```
